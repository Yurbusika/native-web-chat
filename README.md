# FlamiGO chatty

Веб-мессенджер на «нативном» стеке: сервер на Node.js без Express, клиент на ванильном HTML/CSS/JavaScript.

## Основные возможности

- **Регистрация и вход** — сессии в cookie, защищённые страницы и API.
- **Чаты** — список, создание, просмотр, обновление и удаление.
- **Сообщения** — просмотр истории, отправка и удаление сообщений в чате.
- **Поиск пользователей** по имени для создания нового чата.
- **WebSocket** — подключение привязано к сессии (для обновлений в реальном времени на клиенте).
- **Тема оформления** — переключение светлой и тёмной темы на лендинге и в интерфейсе чата.

Данные хранятся в **SQLite**. По умолчанию используется файл `database.db` в корне проекта, создается автоматически при инициализации приложения; путь можно переопределить переменной окружения `DATABASE_PATH`.

## Требования

- [Node.js](https://nodejs.org/) актуальной LTS-версии (в проекте не зафиксировано поле `engines` в `package.json`).

## Установка и запуск

### Бэкенд

1. Клонируйте репозиторий и перейдите в корень проекта.
2. Установите зависимости:

   ```bash
   npm install
   ```

3. Создайте в корне файл `.env` с переменными окружения:

   | Переменная       | Описание |
   | ---------------- | -------- |
   | `PORT`           | Порт HTTP-сервера (например, `3000`). |
   | `HOST`           | Хост для `listen` (например, `localhost`). |
   | `DATABASE_PATH`  | *(необязательно)* путь к файлу SQLite; если не задан — `database.db` в корне. |

   Пример:

   ```env
   PORT=3000
   HOST=localhost
   ```

4. Запуск:

   ```bash
   npm start
   ```

   Команда выполняет `node --env-file=.env --watch server.js`: подхватывается `.env`, сервер перезапускается при изменении отслеживаемых файлов.

В консоли появится сообщение вида: сервер слушает `http://<HOST>:<PORT>`.

### Фронтенд

Отдельного `npm install` для клиента **нет**: разметка, стили и скрипты лежат в каталоге `public/` и отдаются тем же процессом Node после запуска бэкенда.

**Страницы в браузере** (подставьте свой `HOST` и `PORT`):

- `/` — лендинг;
- `/login` — вход;
- `/registration` — регистрация;
- `/chat` — чат (нужна авторизация).

Для правок интерфейса редактируйте файлы в `public/`. Отдельного шага сборки не требуется: достаточно сохранить файлы и обновить страницу в браузере (статика читается с диска при запросе).

## Структура репозитория (кратко)

- `public/` — статический фронтенд (страницы, стили, клиентские модули).
- `auth/`, `chats/`, `messages/`, `user/` — контроллеры и логика API.
- `shared/` — общие утилиты (маршрутизация, БД, cookie, раздача статики).
- `websoket/` — подключение WebSocket к HTTP-серверу.
- `server.js` — точка входа приложения.
